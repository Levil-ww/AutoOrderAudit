问题一、使用后发现会拉取不到待整理订单的详情数据
修复原因：根因：Python 变量绑定陷阱
    问题的本质在 adapter.py 和 material_source.py 中。
    登录后的变量传递链路：
        ① 程序启动 → config.py → AUTHORIZATION = "" (token.json此时为空)
        ② 程序启动 → adapter.py → from .config import AUTHORIZATION
                                  → adapter.AUTHORIZATION = ""  ← 复制了空值！
        ③ 用户登录 → auto_login() → reload_auth()
                                  → config.AUTHORIZATION = "Bearer xxx..." ✅
                                  → adapter.AUTHORIZATION 还是 "" ❌ 没人告诉它
        ④ 点击审单 → FangguoAdapter() → _create_session()
                                       → 用的是 adapter.AUTHORIZATION（空值）
                                       → API认证失败 → 返回空数据

    关键问题：from .config import AUTHORIZATION 在导入时复制变量值，后续 reload_auth() 更新了 config 模块的变量，但 adapter/material_source 模块中的变量还是旧值。

    - 新增 username 字段记录登录账号

新增 reload_auth() 函数，登录成功后调用使新Token立即生效。
