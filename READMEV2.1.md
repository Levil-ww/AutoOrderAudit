# AutoOrderAudit V2.1 — 自动审单系统

基于卖家备注自动解析并修改 ERP 订单商品编码的自动化工具，专为电商客服场景设计。

## 项目概述

AutoOrderAudit 连接方果 ERP 系统，自动拉取待整理订单，解析卖家备注中的**材质-颜色-尺寸-花型**编码，更新订单商品商家编码（shop_mapping_sku），大幅减少客服人工处理订单的时间。

核心流程：

```
方果ERP ──[API]──→ 适配器拉取订单 ──→ 解析卖家备注 ──→ 生成商家编码 ──→ [API]更新回ERP
                                       │
                                       ├─ 自动更新快递（备注关键词/省份规则）
                                       ├─ 补差价订单处理
                                       ├─ 合并订单拆分处理
                                       └─ 赠品检测与编码生成
```

## 功能特性

### 核心功能
- **自动审单** — 批量拉取待整理订单，逐单解析备注并更新编码
- **智能备注解析** — 支持编码格式（`材质-颜色-尺寸-花型`）和自然语言备注的混合解析
- **材质同义词映射** — 内置同义词映射表，兼容客服不同写法（如"双面芊"→"双面格"）
- **材质自动匹配** — 从方果 API 拉取材质列表，支持模糊匹配（`difflib.SequenceMatcher`），识别率高
- **合并订单处理** — 识别合并订单（订单号含 `&` 或多个 `original_tid`），按商品行级备注逐一分配到子订单
- **补差价订单处理** — 自动识别补差价商品行，支持多种场景（不发货、备注含定制信息、混合订单等）
- **赠品检测与编码** — 自动识别备注中的赠品信息，按圆垫/方垫/默认生成对应赠品编码
- **安全关键字过滤** — 备注含"待定"、"等通知发"等关键字时自动跳过
- **多尺寸拆分** — 同一备注中多个尺寸自动拆分为独立商品行，支持材质和花型继承

### V2.1 新增功能
- **自动监控模式** — 自适应间隔轮询待处理订单，有订单时快查（10秒），无订单时逐步拉长至3分钟
- **快递自动配置** — 根据备注关键词或收件省份自动更新订单快递（优先级：备注关键词 > 省份规则）
- **快递配置 GUI** — 可视化编辑省份-快递映射和快递类型
- **Token 自动管理** — 账号密码自动登录，Token 过期自动检测并弹窗提醒重新登录
- **退出登录** — 支持清除 Token 退出登录
- **Dry Run 模式** — 仅打印不提交，安全测试
- **图形界面** — 基于 tkinter 的美化版 GUI，适合非技术人员直接使用

## 技术架构

### 分层设计

| 层 | 模块 | 职责 |
|---|---|---|
| **适配器层** | `adapters/fangguo/` | 实现 `ErpAdapter` 接口，与具体 ERP 系统通信（查单、改码、更新快递） |
| **核心引擎层** | `core/engine.py` | 编排审单流程：合并订单拆分、补差价处理、快递更新、确认回调 |
| **解析引擎** | `core/parser.py` | 通用备注解析：正则提取、多尺寸拆分、材质/花型继承、赠品检测 |
| **认证层** | `auth_client.py` + `auth_manager.py` | Token 自动获取、生命周期管理、过期检测、回调通知 |
| **快递配置层** | `express_config.py` + `express_config.json` | 省份规则、备注关键词映射、快递类型管理 |
| **界面层** | `gui.py` | 桌面图形界面、自动监控、快递配置对话框、Token过期弹窗 |

### 关键抽象

- **ErpAdapter**（`core/adapter_base.py`） — ERP 适配器抽象接口。所有 ERP 系统实现该接口即可接入引擎。
- **Order / OrderItem**（`core/adapter_base.py`） — 通用订单/商品行数据模型，包含 `original_tid`、`shop_remark`、`receiver_province`、`current_cp_code` 等字段。
- **ParsedRemark**（`core/parser.py`） — 备注解析结果模型，包含 `material_code`、`color_code`、`model_code`、`picture_code`、`is_stock`、`gifts`、`original_tid`、`shop_remark` 等字段。
- **AutoAuditEngine**（`core/engine.py`） — 自动审单引擎，遍历订单 → 解析备注 → 更新编码 → 更新快递，支持回调确认、自动监控。

### 处理流程

```
订单进入引擎
  │
  ├─ 检测合并订单（订单号含& 或 多个original_tid）
  │    └─ 按 original_tid 分组，每组使用商品行级备注独立处理
  │
  ├─ 检测补差价订单
  │    ├─ 备注为空/"补差价"/"差价不发货" → 编码改为"定制-定制-补差价-不打印"
  │    ├─ 仅有补差价行 + 备注含定制信息 → 按正常解析修改编码
  │    └─ 混合订单 → 解析结果分配给普通行，剩余分配给补差价行
  │
  ├─ 普通订单处理
  │    ├─ 安全关键字检测（待定、等通知发...） → 跳过
  │    ├─ 解析卖家备注 → 生成编码
  │    ├─ 赠品检测 → 生成赠品行
  │    └─ "赠品不送" → 赠品行编码改为不发货编码
  │
  └─ 更新快递（所有订单通用）
       ├─ 优先：备注关键词匹配（"发极兔"、"发中通"等）
       └─ 其次：收件省份规则匹配
```

## 备注解析规则

### 编码格式

| 类型 | 编码格式 | 示例 |
|---|---|---|
| **定制** | `材质-定制-定制尺寸/裁剪有图-花型;尺寸CM` | `双面格-定制-裁剪有图-花满金陵;100x150CM` |
| **现货** | `材质-标准-尺寸-花型;尺寸`（无cm/CM单位） | `双面格-标准-100x150-花满金陵;100x150` |

### 自然语言格式

支持多种自然语言写法：

- `灰色双面格1张100x150花满金陵`
- `双面格灰色100x150花满金陵1个`
- `圆80cm 双面格 灰色 花满金陵`
- `现货双面革安妮森林;35x160-1张, 40x160-1张`（多尺寸自动拆分）

### 材质自动映射

客服手写材质名称 → 系统标准化编码：

```
双面芊/双面皮革/双面革 → 双面格
pu皮革/PU皮革       → pu防水
镜面革/镜面         → 镜面皮革
真有机硅            → 有机硅
...
```

### 赠品编码规则

| 赠品关键词 | model_code | picture_code |
|---|---|---|
| 圆垫 | 赠品沥水垫小圆或小方 | 赠品沥水垫小圆或小方 |
| 方垫 / 30x50 | 30x50 | 随机发；30x50 |
| 其他（默认） | 赠品沥水垫小圆或小方 | 赠品沥水垫小圆或小方 |

赠品材质统一为 `吸水皮革`，颜色为 `标准`。

### 安全跳过关键字

备注包含以下关键字时自动跳过处理：

```
待定、待确定、待确认、等通知发、需要效果图、效果图、
确认再生产、号发货、安排发货、在安排发货、
已代益、已代森、仓库发、工厂发、补发
```

## 自动监控模式

V2.1 新增自适应间隔轮询功能：

| 参数 | 值 | 说明 |
|---|---|---|
| 有订单时间隔 | 10秒 | 快速处理积压订单 |
| 初始间隔 | 30秒 | 启动后首次检查间隔 |
| 无订单最大间隔 | 180秒（3分钟） | 无订单时逐步拉长 |
| 间隔放大倍数 | 1.5x | 每次无订单时间隔乘以1.5 |

特性：
- 启动时检测已登录状态，自动询问是否开启监控

- 随时可点击「停止监控」关闭
- 与手动审单互斥（手动审单时监控等待）

## 登录与 Token 管理

### 登录方式

- **推荐方式**：通过 GUI 的「登录」按钮，输入手机号密码，程序自动完成二次租户登录
- **手动方式**：技术人员在 `token.json` 中填入 Token 信息

### 登录流程

```
输入手机号+密码
  │
  ├─ 调用 /auth/login 获取初始 Token
  ├─ 调用 /get-permission-info 查找店铺端(type=1) tenant_id
  ├─ 调用 /auth/login/tenant 进行二次租户登录
  └─ 保存最终 Token 到 token.json
```

### Token 文件（`token.json`）

```json
{
  "authorization": "Bearer xxxxx",
  "cookie_str": "JSESSIONID=xxxxx",
  "tenant_id": "3005247",
  "expires_at": "2026-07-18",
  "username": "188xxxx6178",
  "_tips": "===== 此文件由程序自动管理，无需手动修改 ====="
}
```

### Token 生命周期管理

- 自动检测 Token 是否过期（每60秒检查一次）
- 过期前1天提前提醒
- 过期后自动弹出登录框
- 登录成功后自动热加载新 Token（`reload_auth()`）
- 支持退出登录（清除 Token）

## 快速开始

### 环境要求

- Python 3.9+
- 依赖库：`requests`、`urllib3`

### 安装

```bash
# 创建虚拟环境（推荐）
python -m venv .venv

# 激活虚拟环境
# Windows:
.venv\Scripts\activate
# Linux/Mac:
source .venv/bin/activate

# 安装依赖
pip install -r requirements.txt
```

### 运行

**方式一：图形界面（推荐）**

```bash
python run_gui.py
```

首次使用点击「登录」，输入方果 ERP 的手机号密码，程序会自动获取 Token 并保存。登录成功后可选择开启自动监控。

**方式二：命令行**

```bash
python main.py
```

### 配置说明

主要配置在 `adapters/fangguo/config.py` 中：

| 配置项 | 默认值 | 说明 |
|---|---|---|
| `DRY_RUN` | `False` | 设为 `True` 时仅打印不提交，安全测试用 |
| `MAX_ORDERS` | `0` | 最大处理订单数，`0` 表示不限 |
| `MATERIAL_MAP` | （内置映射表） | 材质同义词映射，可自行扩展 |
| `QUERY_STATUS` | `1` | 查询状态，`1`=待整理 |
| `TIME_BEGIN/END` | 最近7天 | 自动计算滚动时间范围 |

## 扩展其它 ERP

如需接入其他 ERP 系统，只需：

1. 在 `adapters/` 下创建新包（如 `jingdong/`）
2. 实现 `ErpAdapter` 接口（`query_orders` + `update_merchant_code`）
3. 可选：实现材质数据源以提供 `material_matcher`
4. 可选：实现 `update_express` 以支持快递更新
5. 引擎和解析器无需改动
